# Lead capture → Supabase (setup, ~5 min)

The contact form (`contact.html`) and the Free Enterprise AI Adoption Consultation
form (`ai-readiness.html`) both submit through `assets/js/leads.js`, which inserts
each submission as a row in a Supabase table called `leads`. You read the leads in
the Supabase dashboard. If Supabase isn't configured yet (or a request fails), the
form automatically falls back to emailing the lead to `hello@alexic.ca`, so nothing
is ever lost.

## 1. Create the project
1. Go to <https://supabase.com> → sign in → **New project** (free tier is fine).
2. Pick a name (e.g. `aic-website`) and a database password (save it). Wait ~2 min for it to provision.

## 2. Create the table + security policy
Open **SQL Editor → New query**, paste this, click **Run**:

```sql
-- leads table
create table if not exists public.leads (
  id            bigint generated by default as identity primary key,
  created_at    timestamptz not null default now(),
  name          text,
  organization  text,
  job_title     text,
  email         text,
  phone         text,
  inquiry_type  text,
  company_size  text,
  message       text,
  source_page   text
);

-- Row Level Security: the public site can ADD leads, but cannot READ them.
alter table public.leads enable row level security;

create policy "anon can insert leads"
  on public.leads
  for insert
  to anon
  with check (true);
```

> Because there is **no SELECT policy**, the public `anon` key can only insert rows,
> never read them. You view leads through the dashboard (which uses your privileged login).

## 3. Get your keys
**Project Settings → API**, copy:
- **Project URL** (looks like `https://abcdxyz.supabase.co`)
- **Project API keys → `anon` `public`** (a long token — this one is safe to ship in the browser)

## 4. Paste them into the site
Open `assets/js/leads.js` and replace the two placeholders near the top:

```js
var SUPABASE_URL      = "https://abcdxyz.supabase.co";   // your Project URL
var SUPABASE_ANON_KEY = "eyJhbGciOiJI...";               // your anon public key
```

Bump the cache version on the `leads.js` `<script>` tags (or hard-refresh) and you're live.

## 5. View your leads
Supabase → **Table Editor → leads**. Every submission appears as a row
(name, organization, job title, email, phone, inquiry type, company size, message,
source page, timestamp). You can sort, filter, and **Export to CSV** from there.

## Notes
- **Free-tier projects pause after ~1 week of inactivity.** If that happens, just log
  into Supabase to wake it; meanwhile submissions still email through to `hello@alexic.ca`.
- **Email alerts:** each lead is also emailed via formsubmit (set `EMAIL_NOTIFY = ""`
  in `leads.js` to turn that off). The first formsubmit email needs a one-time
  confirmation click.
- **Spam:** the form includes a hidden honeypot field. If you get spam later, add
  Supabase rate-limiting or a captcha.
